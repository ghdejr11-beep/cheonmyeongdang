## 3. CODEF 사용 목적 (API 매핑)
| 도메인 | 목적 | CODEF 카테고리 |
|---|---|---|
| 카드 | 사업 비용 자동 집계·분류 | 카드 승인내역/청구서 (본인 카드) |
| 은행 | 입출금 내역, 매출 자동 반영 | 계좌 거래내역 (본인 계좌) |

- CODEF 데이터를 외부에 재판매/중계하지 않음. 내부 경영관리 전용.

## 5. 기술 스택 / 연동 방식
- 서버: Python (FastAPI / 배치 스크립트), Railway 호스팅
- SDK: codef-python / easycodefpy 공식 라이브러리
- 저장: Supabase(PostgreSQL) 또는 로컬 암호화 저장
- 호출 주기: 1일 1회 심야 배치 (실시간 호출 없음)
- Connected ID: 본인 계정 기반 다기관 연결

## 6. 보안 계획
1. 키 관리: client_id/client_secret/public key는 .env + .gitignore, Railway Secret Manager에 저장. 코드 하드코딩 금지.
2. 데이터: 원문 저장 최소화, 집계 결과만 보존. 저장 시 AES 암호화.
3. 접근: 대표자 1인 접근, 2FA 활성화, IP 화이트리스트.
4. 동의: 본인 계정 전용 → 자기 동의로 커버, 제3자 수집 없음.
5. 사고 대응: 키 유출 의심 시 즉시 재발급, support@codef.io 30분 내 통보.
