## 2. 서비스 목적

사용자가 여러 보험사에 흩어져 있는 보험 가입 현황을 **한 곳에서 통합 조회**하고, AI 분석을 통해 **중복 보장 제거**, **누락 보장 발견**, **맞춤 보험 추천**을 받을 수 있도록 합니다.

---

## 3. 주요 기능

### 3.1 내보험다보여 API 연동 (34개 보험사)
- 금융감독원 내보험다보여 API를 통해 사용자의 전체 보험 가입 내역 조회
- 생명보험, 손해보험, 공제 등 34개 보험사 통합 지원
- 보험 계약 상세 정보: 보험명, 보장내용, 납입현황, 만기일 등

### 3.2 마이헬스웨이 진료기록 연동
- 보건복지부 마이헬스웨이 API를 통해 사용자 진료기록 조회
- 진료 이력 기반 건강 리스크 분석
- 현재 보험 보장과 실제 건강 상태 매칭 분석

### 3.3 가입 전 건강분석
- 사용자 건강 데이터 기반 보험 가입 적합도 사전 분석
- AI가 현재 건강 상태에 맞는 보험 상품 유형 추천
- 기존 가입 보험과의 중복/누락 보장 자동 분석

### 3.4 약관 연결
- 각 보험 상품별 약관 원문 링크 제공
- 주요 보장 항목 요약 (AI 요약)
- 보장 범위 비교 기능

### 3.5 설계사 연결
- 지역/전문 분야별 보험 설계사 매칭
- 인앱 상담 신청 기능
- 설계사 평점/리뷰 시스템

## 4. 화면 구성 (스크린샷 설명)

### 화면 1: 메인 대시보드
- 상단: 사용자 이름, 총 가입 보험 수, 월 납입 총액 표시
- 중앙: 보험사별 가입 현황 카드 리스트 (로고 + 보험명 + 월 보험료)
- 하단: 내비게이션 바 (홈 / 분석 / 약관 / 설계사 / 설정)

### 화면 2: AI 보장 분석
- 보장 영역별 분석 결과 (사망, 입원, 수술, 진단, 실손 등)
- 중복 보장 영역 빨간색 경고 표시
- 누락 보장 영역 노란색 권고 표시
- 하단: "맞춤 보험 추천 받기" 버튼

### 화면 3: 건강 분석 리포트
- 마이헬스웨이 연동 진료 기록 요약
- 건강 리스크 점수 (AI 산출)
- 현재 보험 보장 vs 건강 리스크 매칭 그래프

### 화면 4: 설계사 연결
- 추천 설계사 프로필 카드
- 전문 분야, 경력, 평점 표시
- "상담 신청" 버튼

## 5. 데이터 활용 계획

### 5.1 데이터 흐름
```
사용자 본인인증 → 내보험다보여 API 호출 → 보험 가입 현황 수신
    ↓
마이헬스웨이 API 호출 → 진료기록 수신
    ↓
AI 분석 엔진 → 중복/누락 보장 분석 + 건강 리스크 매칭
    ↓
맞춤 보험 추천 결과 생성 → 사용자에게 표시
```

### 5.2 활용 목적
1. **보험 가입 현황 조회**: 사용자가 자신의 전체 보험을 한눈에 확인
2. **맞춤 추천**: AI가 사용자 상황에 맞는 보험 상품 유형 추천
3. **중복/누락 분석**: 불필요한 중복 보장 제거, 필요한 보장 누락 알림
4. **건강 기반 분석**: 진료기록과 보험 보장을 교차 분석하여 실질적 보장 적합도 평가

### 5.3 데이터 보관 정책
- 조회 데이터는 분석 완료 후 사용자 단말에만 저장 (서버 미보관)
- 사용자 요청 시 즉시 삭제 가능
- 분석 결과는 익명화 처리 후 서비스 개선 목적으로만 활용
